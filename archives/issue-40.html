TIP OF THE WEEK

PostgreSQL : Use a ~/.pgpass file to avoid typing passwords

Contributed by Fabien Loudet, Linux SysAdmin at Rosetta Stone

Password-less authentication in Postgres can be achieved by putting a ~/.pgpass file in a userâ€™s home directory.

# The format to use for each line in the file: hostname:port:database:username:password

# Only the owner of the file should have read and write access to it: chmod 0600 ~/.pgpass

# Enjoy password-less authentication! $ psql -h hostname -U username -d database # As long as the connection parameters are properly set in # the .pgpass file, you should not be prompted for password
# BONUS: Use password-less authentication on # backup scripts with pg_dump and its -w option: $ pg_dump -h hostname -U username -d database -w > dump.bak
